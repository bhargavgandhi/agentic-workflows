# Branch Naming Conventions

## Format
`<type>/<short-description>`

## Guidelines
- `feature/`: For implementing new features.
  - Example: `feature/user-dashboard`
- `fix/`: For bug fixes.
  - Example: `fix/login-timeout`
- `hotfix/`: For critical fixes directly to production or main branches.
  - Example: `hotfix/billing-crash`
- `chore/`: For maintenance, dependency bumps, etc.
  - Example: `chore/update-typescript`

Avoid capitalization, underscores, or spaces. Use lowercase alphanumeric and hyphens.
