# RTK Query Endpoint Patterns

## Base API
- Use a central `api.ts` initialized by `createApi()`.
- Define `baseQuery` (e.g. `fetchBaseQuery({ baseUrl: '/api' })`).
- Use `injectEndpoints` to add endpoints dynamically from feature-specific files.

## Queries
- Use `query` endpoints for read operations.
- Map responses to strictly typed TypeScript interfaces.
- Specify `providesTags`.

## Mutations
- Use `mutation` endpoints for create, update, delete operations.
- Specify `invalidatesTags`.
- Structure optimistic updates using `onQueryStarted` if immediate feedback is needed.

## Generated Hooks
- Rely completely on autogenerated hooks (`useGetPostsQuery`, `useUpdatePostMutation`) instead of querying manually.
